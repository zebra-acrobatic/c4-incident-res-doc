KQL Commands

Essential Commands for Data Analysis
summarise • extend • order • top • render

Interactive Learning Guide for Students

What is KQL?

Kusto Query Language (KQL) is a powerful tool for analysing large datasets.

  • Used in Microsoft Azure for log analytics
  • Similar to SQL but optimised for big data analysis
  • Perfect for finding patterns in large amounts of data
  • Uses a "pipe" (|) approach to chain commands together
TableName | command1 | command2 | command3

Commands We'll Learn Today

summarise

Groups data and calculates totals, averages, counts

extend

Adds new calculated columns to your data

order

Sorts your results ascending or descending

top

Gets the highest or lowest values

render

Creates bar charts from your data

The summarise Command

Groups data and performs calculations - like creating a summary table in Excel.

Basic Syntax:

TableName | summarise AggregateFunction(Column) by GroupingColumn

Common Functions:

  • count() - counts rows
  • sum() - adds up values
  • avg() - calculates average
  • max() / min() - finds highest/lowest

summarise Examples

Example 1: Website visits by country
WebTraffic | summarise VisitCount = count() by Country
Shows how many visits came from each country
Example 2: Average response time by server
ServerLogs | summarise AvgResponseTime = avg(ResponseTimeMs) by ServerName
Shows average response time for each server

The extend Command

Adds new calculated columns without changing existing data - like adding a formula column in a spreadsheet.

Basic Syntax:

TableName | extend NewColumnName = Calculation

Key Point: extend doesn't remove any existing columns, it just adds new ones!

extend Examples

Example 1: Temperature conversion
WeatherData | extend TempFahrenheit = (TempCelsius * 9/5) + 32
Example 2: Creating categories
ServerMetrics | extend StatusCategory = case( CPUUsage > 90, "Critical", CPUUsage > 70, "High", CPUUsage > 50, "Medium", "Normal" )

The order Command

Sorts your results in ascending or descending order.

Basic Syntax:

TableName | order by ColumnName asc/desc
  • asc = ascending (lowest to highest)
  • desc = descending (highest to lowest)
  • Can sort by multiple columns

order Examples

Example 1: Most recent logs first
ApplicationLogs | order by Timestamp desc
Example 2: Multiple columns
StudentGrades | order by Subject asc, Grade desc
Sorts by subject alphabetically, then by highest grade within each subject

The top Command

Returns the highest or lowest values - perfect for "top 10" lists!

Basic Syntax:

TableName | top NumberOfRows by ColumnName asc/desc
  • desc gives you the highest values
  • asc gives you the lowest values
  • Much more efficient than using order + take

top Examples

Example 1: Top 5 cities by population
CityData | top 5 by Population desc
Example 2: Most common error types
ErrorLogs | summarise ErrorCount = count() by ErrorType | top 10 by ErrorCount desc

The render Command

Creates visualisations from your data - we'll focus on bar charts.

Basic Syntax:

TableName | render barchart
  • Bar charts are excellent for comparing categories
  • Works best with grouped/summarised data
  • Always put render as your last command

render barchart Examples

Example 1: Sales by region
RegionalSales | summarise TotalSales = sum(SalesAmount) by Region | render barchart
Example 2: Monthly website traffic
WebsiteMetrics | extend Month = format_datetime(Date, "yyyy-MM") | summarise PageViews = sum(Views) by Month | order by Month asc | render barchart

Combining Commands

The real power of KQL comes from chaining commands together!

Top 5 Australian cities by average temperature
WeatherReadings | where Country == "Australia" | extend TempFahrenheit = (TempCelsius * 9/5) + 32 | summarise AvgTemp = avg(TempFahrenheit) by City | top 5 by AvgTemp desc | render barchart

Notice how each command builds on the previous one!

More Complex Examples

Server performance analysis
ServerPerformance | extend PerformanceCategory = case( ResponseTime > 1000, "Slow", ResponseTime > 500, "Average", "Fast" ) | summarise ServerCount = count() by PerformanceCategory | order by ServerCount desc | render barchart

Common Query Patterns

Pattern 1: Analysis with visualisation

DataTable | summarise Metric = aggregation(Column) by Category | top N by Metric desc | render barchart

Pattern 2: Time-based analysis

DataTable | extend TimeGroup = format_datetime(DateColumn, "format") | summarise Aggregation = function(Value) by TimeGroup | order by TimeGroup asc | render barchart

Best Practices

💡 Key Tips for Success

  • Start small: Use | take 100 whilst learning
  • Use meaningful names: VisitorCount = count() not just count()
  • Order your operations: Filter → Extend → Summarise → Order → Render
  • Test as you go: Run each step to check it works
  • Bar charts for categories: Perfect for comparing different groups

Practice Exercise

Challenge: Create a query that shows the top 3 product categories by average sale price, displayed as a bar chart.

Given table: ProductSales

Columns: ProductCategory, SalePrice, Quantity

Think about:

  • Which commands do you need?
  • What order should they go in?
  • What should you call your calculated column?

Exercise Solution

Top 3 product categories by average sale price
ProductSales | summarise AvgSalePrice = avg(SalePrice) by ProductCategory | top 3 by AvgSalePrice desc | render barchart

Breaking it down:

  1. summarise - Calculate average sale price for each category
  2. top 3 - Get the 3 highest values
  3. render barchart - Display as a bar chart

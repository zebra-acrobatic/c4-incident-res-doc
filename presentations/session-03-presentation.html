What is SQL?

SQL (Structured Query Language) is a standard language for managing relational databases. It's been around since the 1970s and is used everywhere!

Key Concepts:

  • Tables: Data organized in rows and columns
  • Rows: Individual records
  • Columns: Data fields/attributes
  • Relationships: How tables connect to each other

Sample Employee Table:

ID Name Department Salary
1Alice JohnsonEngineering75000
2Bob SmithMarketing65000
3Carol DavisEngineering80000
4David WilsonSales70000

Basic SQL Operations

1. SELECT - Getting Data

Get all employees.

SELECT * FROM employees;

Get specific columns.

SELECT name, department FROM employees;

2. WHERE - Filtering Data

Find all the engineers.

SELECT * FROM employees WHERE department = 'Engineering';

Find high earners.

SELECT name, salary FROM employees WHERE salary > 70000;

3. ORDER BY - Sorting Data

Sort by salary (highest first).

SELECT name, salary FROM employees ORDER BY salary DESC;

More SQL Power!

Aggregation Functions

Count employees by department.

SELECT department, COUNT(*) as employee_count FROM employees GROUP BY department;

Create and average salary based on the salary listings.

SELECT AVG(salary) as average_salary FROM employees;

Joining Tables

Join employees with their projects using the projects table and the employes table.

SELECT e.name, p.project_name FROM employees e JOIN projects p ON e.id = p.employee_id;
üí° SQL Summary: SQL is great for structured data in traditional databases. It uses tables with predefined schemas and focuses on relationships between data.

Enter KQL: Kusto Query Language

KQL (Kusto Query Language) is Microsoft's query language for analysing large amounts of log and telemetry data. It's optimised for time-series data and analytics.

Where You'll Find KQL:

  • Azure Monitor: Application and infrastructure monitoring
  • Microsoft Sentinel: Security information and event management
  • Application Insights: Application performance monitoring
  • Azure Data Explorer: Big data analytics platform

SQL Focus

  • Structured relational data
  • CRUD operations
  • Transactions
  • Normalized tables

KQL Focus

  • Time-series & log data
  • Analytics & insights
  • Real-time queries
  • Semi-structured data

KQL Fundamentals

KQL uses a pipeline approach - you start with data and pipe it through operators.

Basic Syntax Pattern:

Basic structure.

TableName | operator1 | operator2 | operator3

Sample Log Data (Application Insights):

timestamp operation_Name duration resultCode
2024-01-15T10:30:00GET /api/users150ms200
2024-01-15T10:31:00POST /api/login300ms200
2024-01-15T10:32:00GET /api/users2000ms500
2024-01-15T10:33:00GET /api/orders100ms200

Essential KQL Operations

1. where - Filtering (like SQL WHERE)

Find slow requests.

requests | where duration > 1000

Find errors in the last hour.

requests | where timestamp > ago(1h) and resultCode >= 400

2. project - Selecting columns (like SQL SELECT)

Select specific columns.

requests | project timestamp, operation_Name, duration

3. summarize - Aggregation (like SQL GROUP BY)

Count requests by result code.

requests | summarize count() by resultCode

Average duration by operation.

requests | summarize avg(duration) by operation_Name

KQL vs SQL: Side by Side

SQL Version

Get error count by hour.

SELECT DATEPART(hour, timestamp) as hour, COUNT(*) as error_count FROM requests WHERE resultCode >= 400 AND timestamp > DATEADD(day, -1, GETDATE()) GROUP BY DATEPART(hour, timestamp) ORDER BY hour;

KQL Version

Get error count by hour.

requests | where resultCode >= 400 | where timestamp > ago(1d) | summarize error_count = count() by bin(timestamp, 1h) | sort by timestamp
